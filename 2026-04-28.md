### 1. rx_970 實驗結果分析與報告

比較 `rx_970_cycu.pcap`（shared_jos v1）與 `rx_970_cycu_no_edit.pcap`（shared_jos v0）：

- **封包格式差異**：v0 = Ethernet(14B)+IPv4(20B)+comp×4，v1 = 純 comp×4（56B），每封包節省 34 bytes（37.8%）
- **Tuple 錯位問題**：v1 輸出整體向後偏移 2 個 tuple
- **根本原因**：`packet_count` register 在測試前已有殘留狀態（背景 HTTP 流量 `61.145.207.213:80↔123.38.204.24:42110` 先到 2 筆，將 register 停在位置 2）
- **套用 +2 offset 後比對結果**：IP/port 完全一致率 0% 錯誤，全程 match（89.5% 完全一致，10.5% 為 DNS 雙協定差異）

#### 位移示意

```
v0:  [ T1 ][ T2 ][ T3 ][ T4 ][ T5 ][ T6 ] ...
                   ▲      ▲      ▲      ▲
v1:  [BG1 ][BG2 ][ T1 ][ T2 ][ T3 ][ T4 ] ...

第一個 comp 封包（v1）：
  comp_0 = BG1 (61.145.207.213:80→123.38.204.24:42110) ← 殘留背景流量
  comp_1 = BG2 (123.38.204.24:42110→61.145.207.213:80)
  comp_2 = T1  (正式流量第 1 筆)
  comp_3 = T2  (正式流量第 2 筆)
```

### 2. rx_970_cycu_v2_send / rec 分析

- **send**（1001 pkt）：Tofino 輸入端完整捕捉，含測試流量 + 背景 HTTP 流量混雜
- **rec**（248 pkt）：壓縮後輸出，comp-only 格式
- **發現**：rec 頭部的 BG 資料來自 send capture 開始**之前**已存在 register 的舊值（非「send 尾端流量跑到 rec 頭端」）
  - rec pkt#1 輸出時間：+0.003s
  - BG 流量在 send 首次出現：+0.080s → 因果順序確認為 register 殘留

### 3. shared_jos_v2 build error 修復

bf-p4c 9.7.0 PHV constraint analyzer 的 compiler bug：

| 嘗試方案 | 結果 |
|----------|------|
| `@defaultonly NoAction` + `const default_action = NoAction` | 失敗 |
| 完全移除 NoAction | 失敗 |
| 自定義 `nop()` action | 失敗 |
| **移除整個 `comp_reset` table** | **成功** |

最終解法：移除 data plane 的 `comp_reset` table，register 清零改用 bfrt_python 直接寫：
```python
bfrt.switch_tofino2_y1.pipe.SwitchIngress.COMP.packet_count.entry_mod(REGISTER_INDEX=0, f1=0)
```

### 4. shared_jos_v2 my_comp_egress.p4 修改（4:1 輸出）

實現 1000 封包進、250 封包出（4:1 比例），不使用 mirror：

**改動：**
- 非 comp 封包（count 0,1,2）→ `eg_intr_md_for_dprsr.drop_ctl = 1`（直接丟棄）
- comp 封包（count 3）→ `setInvalid()` 剝除 Ethernet/IPv4，輸出純 56B comp

移除不再需要的：`flag` register、`flag_up`/`flag_down`、`mirror_fwd` table

**bfrt command.py 簡化為：**
```python
bfrt.switch_tofino2_y1.pipe.SwitchIngress.comp.ipv4_set_port.entry_with_set_port(valid=1, port=442).push()
```

### 5. miura_shared_jos_v2 建立

複製 `miura_shared_jos` → `miura_shared_jos_v2`，套用相同邏輯（9.13.2 版本用 `mtu_trunc_len = 56` 取代 `setInvalid()`）：

**my_comp.p4**
```p4
// hdr.ipv4.src_addr = 0x0a000001;  // 關閉自訂 IP
// hdr.ipv4.dst_addr = 0x0a000002;
```

**my_comp_egress.p4**
```p4
apply {
    if (!hdr.comp_0.isValid()) {
        eg_intr_md_for_dprsr.drop_ctl = 1;       // 丟棄非 comp
    } else {
        eg_intr_md_for_dprsr.mtu_trunc_len = 56; // 截斷至 comp_0~3
    }
}
```

移除：`flag` register、`flag_up`/`flag_down`、`mirror_fwd` table（與 shared_jos_v2 相同邏輯，API 不同）
